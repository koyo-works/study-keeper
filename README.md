## サービス概要 
「study-keeper」は、つい他のことをしてしまう人向けの行動選択支援アプリです。
無意識の行動を意識的に選択することで、学習や自己管理習慣の定着をサポートします。

## このサービスへの思い・作りたい理由
自分自身、勉強しようと思ってもゲームや動画、SNSに気を取られてしまい、気づけば1日が終わっていた経験があります。
「やる気はあるのに続かない」原因の多くは、無意識に行っている行動の積み重ねにあると感じています。

そこで、行動を“選ぶ”という小さな意識の変化から、習慣を整えるアプリを作りたいと思いました。
このアプリを通じて、同じように「続けたいのに続かない人」が少しでも前に進めるきっかけを作れればと思っています。

## ユーザー層

### メインターゲット
##### Webエンジニアを目指して学習中のスクール生・独学者
 - 「やる気はあるのに継続が難しい」という共通の悩みを抱える
 - 自由度が高く学習内容も多いため自己管理能力が求められる
 - 学習中に誘惑に負けやすく、モチベーションが上下しやすい

##### 学習・資格勉強をしている社会人・学生
 - 仕事や学校の合間に勉強時間を確保する必要がある
 - 行動を「選ぶ」という軽い操作で学習意識を切り替えられる設計が、心理的ハードルを下げる
 - SNS・動画などに気を取られる現代の生活習慣にマッチ

## サービス利用イメージ
- アプリを開くと「今何をしようとしている？」という質問が表示
  - 例：「勉強する」「SNSを見る」「動画を見る」「一服」
- 行動をボタンで選ぶと記録され、日や週ごとの傾向を自動で可視化
  - 例：「今週は勉強を選んだ回数が先週より20%増えた」
- 選択データを基に、曜日・時間帯別や月間の傾向グラフも確認可能
  - 自分の意思決定パターンや誘惑に負けやすい時間帯を客観的に把握
- 習慣化サポート機能で、連続日数（ストリーク）や日・週の振り返りメッセージを確認
  - 「今日は○日連続で勉強を選択できた！」など達成感を演出
- 将来的にはポイント・レベルアップ・バッジなどのゲーミフィケーション要素でモチベーション向上
- チーム・ソーシャル機能により、匿名比較や励ましメッセージを通じて継続意欲を強化
- 結果として、無意識のサボりを減らし、意識的に勉強に向かう習慣を身につけられる
- 勉強の継続率向上と自己管理力の強化という価値をユーザーに提供

## ユーザーの獲得について
- 学習者コミュニティ（例：RUNTEQ生、プログラミング学習者）への発信
  - Times・Discord・Xで「学習習慣を崩しがちな人向けの自己管理ツール」として共有
- SNSでの発信
  - 「今日も勉強サボった」など共感されやすい悩みを投稿し興味を持った人にサービスを知ってもらう

## サービスの差別化ポイント・推しポイント
### 意思決定フォーカス
- 一般的な習慣化アプリは「やった／やらなかった」結果を記録する
- 本サービスは「行動の選択の瞬間」にフォーカス
  - 例：「SNSを見るか勉強するか」を意思決定の段階で記録・可視化
  - 自分の“選び方のクセ”を知ることができる

### 思考プロセス可視化
- 記録対象は“行動ログ”ではなく“選択ログ”。  
- 「SNSを見ようとしたけど勉強を選んだ」など、**思考プロセスそのものをデータ化**。  
- これにより、自己コントロール力や誘惑に負けるタイミングを客観的に把握できる。

### 行動を促すUX
- 通知やシンプルなUIで「今何してる？」と問いかけ、アプリを開くきっかけを作る
- 将来的にはウィジェット・ブラウザ通知などを実装予定

### 前向き選択の評価
- 「勉強を選んだ回数」「連続選択日数」を表示
- 選択できた日を記録対象とし、罪悪感のない習慣化を実現

### 汎用性
- 勉強・SNS以外の選択肢も自由に設定可能
- 学習者以外でも、自己管理・時間意識向上に応用可能

## 機能候補

### MVP（卒業制作向け）
- 目的：意思決定そのものを記録・可視化する最小限機能

#### 機能
1. ユーザー認証（新規登録 / ログイン / ログアウト）
2. 行動選択の記録
   - 選択肢例：「勉強する」「SNSを見る」「動画を見る」「一服」
   - ボタンクリックで記録
   - 任意メモ入力（オプション）
3. 記録データの可視化
   - 今日の選択回数
   - 今週の選択割合（棒グラフ or 円グラフ）
4. 習慣化サポート
   - ストリーク（連続日数）表示
   - 軽いプッシュ通知（1日1〜2回）
5. 簡易トップページ
   - アプリ概要＋「今すぐ始める」ボタン
